# Архитектура Монорепозитория (Bun Workspaces)

Этот проект — монорепозиторий. Код разделен на независимые пакеты в папке `apps/`. 
Ты ОБЯЗАТЕЛЬНО должен учитывать контекст пакета, в котором редактируешь файл. Смешивание сред ЗАПРЕЩЕНО.

## Пакеты (Workspaces)

### 1. `apps/client` (Frontend)
- **Стек:** Vue 3 (Composition API), Vite.
- **Среда:** Браузер.
- **Правило:** Здесь ЗАПРЕЩЕНО использовать Bun API (например, `Bun.serve`, `Bun.file`), обращаться к базе данных напрямую или использовать пакеты Node.js (fs, path). Все запросы к серверу идут только через API-клиент.

### 2. `apps/server` (Backend)
- **Стек:** Bun, TypeScript.
- **Среда:** Сервер (Bun runtime).
- **Правило:** Здесь НЕТ браузерных API (`window`, `document`, `localStorage`). Вся работа с файловой системой и сетью должна использовать нативные API Bun, где это возможно. Никакого Vue/UI кода.

### 3. `apps/*-parser` (Парсеры: en-dict, jp-dict, manhuagui, zh-dict)
- **Стек:** Bun скрипты / CLI утилиты.
- **Назначение:** Парсинг данных из внешних источников.
- **Правило:** Это изолированные утилиты. Они не должны зависеть от Vue-клиента.

# Обязательные действия (Вместо хуков)

1. Перед завершением работы и написанием ответа ВСЕГДА запускай команду `bun run lint:fix`.
2. После этого ВСЕГДА запускай команду `bun run typecheck`.
3. Если есть ошибки типов или линтера — исправь их самостоятельно.
