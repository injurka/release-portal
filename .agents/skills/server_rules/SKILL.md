# Архитектура Backend (Server)

## Работа с Базой Данных (Drizzle + SQLite)
1. **Схема:** Вся структура БД описана в `apps/server/src/db/schema.ts`. Если тебе нужно написать новый запрос или создать таблицу, СНАЧАЛА прочитай этот файл.
2. **Анализ данных (MCP):** Если тебе нужно понять, какие реально данные лежат в базе (например, посмотреть формат JSON в колонке), используй MCP-инструмент `sqlite-main-db`, чтобы сделать `SELECT`. 
3. **ORM:** Запрещено писать сырые SQL строки (кроме `sql\`...\`` из Drizzle). Всегда используй методы Drizzle (`db.select()`, `db.insert()`, `db.query.tableName.findFirst()`).

## Инструменты и Среда (Bun)
- Сервер работает на Bun. ЗАПРЕЩЕНО использовать стандартные модули Node.js, если есть нативные, более быстрые аналоги Bun (например, используй `Bun.serve`, `Bun.file`, `Bun.password`). ЗАПРЕЩЕНО использовать MCP-инструменты `vue-docs`, так как они относятся только к фронтенду.
- Если ты сомневаешься в синтаксисе API Bun, **НЕ ГАДАЙ!**
  1. Сначала используй MCP-инструмент `search_bun` для поиска нужной темы.
  2. Чтобы прочитать конкретную страницу, используй инструмент `query_docs_filesystem_bun` (например, вызови команду `cat /api-reference/file.mdx` или `rg "keyword" /`).
